- 무엇을: weekly market report의 운영 안정성과 본문 가독성을 개선했다.
- 왜: 실제 로그에서 KRX DNS 실패와 빈 데이터 예외가 있었고, 메일 본문도 표 나열 중심이라 핵심 변화와 fallback 상태를 바로 읽기 어려웠다. 추가로 수동 실행 시 외부 DNS 실패로 SMTP 송신까지 검증이 막혀, 송신 실패 시에도 로컬 산출물을 남기는 장치가 필요했다.
- 어떻게: `weekly_report.py`에 안전한 표 렌더링과 주간 요약 섹션을 추가하고, S&P 500 캐시를 stale fallback까지 활용하도록 보강했다. 이후 `--dry-run`, `--offline`, `outputs/` 산출물 저장, SMTP 실패 시 HTML/TXT/JSON 저장 fallback을 추가했고, 각 실행마다 `history/*.csv`에 시장별 변동 기록이 누적되도록 만들었다. 또 Yahoo bulk download에서 비어 버린 미국 종목은 개별 재시도하도록 보강하고, alert 누락 메시지도 "이번 실행에서 가격 데이터가 없었다"는 의미로 수정했다. `.env.example`와 README 실행 가이드도 보완했다.
- 결과: 데이터 일부가 비어도 리포트 생성이 유지되고, 상단 요약에서 시장별 강약세와 alert 트리거 현황, fallback 상태를 바로 확인할 수 있게 됐다. 네트워크 또는 SMTP 실패 시에도 생성된 리포트를 로컬 파일로 검토할 수 있고, 비정기 실행이더라도 `history/` 폴더에서 종목별 변동 흐름을 시간순으로 추적할 수 있다.
